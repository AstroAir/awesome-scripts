# ❓ 常见问题

本文档收集了用户最常遇到的问题及解决方案。

## 安装问题

### Q: 点击 .user.js 文件没有弹出安装界面？

**A:** 请确认已正确安装用户脚本管理器：

1. 检查是否已安装 Tampermonkey 或 Violentmonkey
2. 确认扩展已启用
3. 尝试点击文件后再点击 "Raw" 按钮
4. 如果还是不行，复制脚本内容手动创建新脚本

### Q: 安装后脚本不生效？

**A:** 尝试以下步骤：

1. **刷新页面** - 最简单的解决方法
2. **检查脚本状态** - 确认脚本已启用
3. **检查匹配规则** - 确认当前网址匹配脚本的 `@match` 规则
4. **查看控制台** - 按 F12 查看是否有错误信息
5. **重新安装** - 删除脚本后重新安装

### Q: 如何更新脚本？

**A:** 有几种方式：

1. **自动更新** - Tampermonkey 会定期检查更新
2. **手动检查** - 在 Tampermonkey 中点击"检查更新"
3. **重新安装** - 访问脚本页面重新安装

## GitHub 脚本问题

### Q: Trending 按钮不显示？

**A:**

1. 确认在 GitHub 页面（github.com）
2. GitHub 使用 SPA 架构，尝试刷新页面
3. 检查是否有其他扩展冲突
4. 查看控制台是否有错误

### Q: 文件折叠功能失效？

**A:**

1. 确认在仓库的文件列表页面
2. GitHub 更新后选择器可能变化，请更新到最新版本
3. 尝试刷新页面

### Q: 工具栏按钮样式异常？

**A:**

1. 清除浏览器缓存
2. 检查是否与其他 GitHub 相关扩展冲突
3. 尝试切换 GitHub 主题后刷新

### Q: 按钮点击后没有反应？

**A:**

1. 确认当前页面是仓库页面（有 owner/repo 路径）
2. 部分服务可能需要仓库是公开的
3. 检查网络连接

## Linux.do 脚本问题

### Q: 自动阅读脚本不工作？

**A:**

1. 确认已登录 Linux.do 账号
2. 确认在正确的页面（帖子列表或帖子详情）
3. 检查是否被论坛限制（暂时降低速度）
4. 尝试使用更慢的模式（仿真/隐身）

### Q: 控制面板无法拖动？

**A:**

1. 确认鼠标在面板标题栏区域拖动
2. 检查是否有其他元素遮挡
3. 尝试刷新页面

### Q: 进度显示不准确？

**A:**

1. 进度基于已处理的帖子数量估算
2. 实际进度可能因网络延迟有偏差
3. 以实际处理的帖子数为准

### Q: 导出的文件打不开？

**A:**

1. **JSON 文件** - 使用文本编辑器或 JSON 查看器打开
2. **HTML 文件** - 使用浏览器打开
3. 如果文件为空，可能导出过程出错，请重试

### Q: 导出的图片显示不了？

**A:**

1. 如果没有选择"嵌入图片"，需要联网查看
2. 选择"嵌入图片"选项重新导出
3. 部分图片可能有防盗链，无法嵌入

## 开发问题

### Q: pnpm install 失败？

**A:**

1. 确认 Node.js 版本 >= 18
2. 确认 pnpm 版本 >= 8
3. 删除 `node_modules` 和 `pnpm-lock.yaml` 后重试
4. 检查网络连接，可能需要设置代理

### Q: 构建报错？

**A:**

1. 运行 `pnpm lint` 检查代码问题
2. 确认所有导入路径正确
3. 检查是否有语法错误
4. 查看完整错误信息定位问题

### Q: 开发模式下脚本不更新？

**A:**

1. 确认 `pnpm dev` 正在运行
2. 确认安装的是 `.proxy.user.js` 代理脚本
3. 手动刷新页面
4. 检查文件保存是否触发了重新构建

### Q: 测试失败？

**A:**

1. 运行 `pnpm test` 查看详细错误
2. E2E 测试需要先安装浏览器：`npx playwright install`
3. 检查测试环境配置是否正确
4. 部分测试可能依赖网络环境

## 兼容性问题

### Q: 支持哪些浏览器？

**A:** 脚本支持所有现代浏览器：

- Chrome 80+
- Firefox 75+
- Edge 80+
- Safari 14+

### Q: 支持移动端吗？

**A:**

目前主要针对桌面端设计。部分功能在移动端可能无法正常使用。

### Q: 与其他扩展冲突？

**A:**

如果发现冲突：

1. 暂时禁用其他扩展排查
2. 在 Issue 中报告冲突的扩展
3. 我们会尽量解决兼容性问题

## 其他问题

### Q: 如何反馈 Bug？

**A:**

请在 [GitHub Issues](https://github.com/AstroAir/awesome-scripts/issues) 提交，包含：

1. 脚本名称和版本
2. 浏览器和扩展版本
3. 问题描述和复现步骤
4. 控制台错误截图（如有）

### Q: 如何请求新功能？

**A:**

同样在 GitHub Issues 提交，标注为 Feature Request，说明：

1. 功能描述
2. 使用场景
3. 期望的实现方式

### Q: 如何贡献代码？

**A:**

请参考 [贡献指南](../CONTRIBUTING.md)，简要流程：

1. Fork 仓库
2. 创建功能分支
3. 开发和测试
4. 提交 Pull Request

### Q: 脚本会收集用户数据吗？

**A:**

**不会**。所有脚本：

- 不收集任何用户数据
- 不发送任何数据到第三方服务器
- 所有数据都存储在本地浏览器中
